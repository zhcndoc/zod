## 介绍

Zod 是一个 TypeScript 优先的验证库。使用 Zod，你可以定义*模式*来验证数据，从简单的 `string` 到复杂的嵌套对象。

```ts
import { z } from "zod/v4";

const User = z.object({
  name: z.string(),
});

// 一些不可信的数据...
const input = { /* 内容 */ };

// 解析结果是验证过且类型安全的！
const data = User.parse(input);

// 因此你可以放心使用 :)
console.log(data.name);
```

## 特性

- 零外部依赖
- 支持 Node.js 和所有现代浏览器
- 极简：核心包大小 2kb（压缩后）
- 不可变 API：方法返回新实例
- 简洁接口
- 支持 TypeScript 和普通 JS
- 内置 JSON Schema 转换
- 丰富的生态系统


## 安装

{/* ### `npm` */}

```sh
npm install zod       # npm
deno add npm:zod      # deno
yarn add zod          # yarn
bun add zod           # bun
pnpm add zod          # pnpm
```

Zod 每次提交都会发布一个 canary 版本。安装 canary 版本：

```sh
npm install zod@canary       # npm
deno add npm:zod@canary      # deno
yarn add zod@canary          # yarn
bun add zod@canary           # bun
pnpm add zod@canary          # pnpm
```

## 要求

Zod 在 *TypeScript v5.5* 及更高版本中测试通过。较旧版本可能可用，但不正式支持。

### `"strict"`

你必须在 `tsconfig.json` 中启用 `strict` 模式。这是所有 TypeScript 项目的最佳实践。
  
```ts
// tsconfig.json
{
  // ...
  "compilerOptions": {
    // ...
    "strict": true
  }
}
```

### `"moduleResolution"`

你的 `"moduleResolution"` 应设置为以下之一。旧的 `"node"` 和 `"classic"` 模式不支持，因为它们不支持子路径导入。

- `"node16"`（如果 `"module"` 设置为 `"node16"`/`"node18"`，默认使用）
- `"nodenext"`（如果 `"module"` 设置为 `"nodenext"`，默认使用）
- `"bundler"`
